“a modelling technology and associated set of processes to produce, communicate, and analyze building models.”
So BIM is not 3D - it is a set of technologies and methodologies related to information modeling in the building domain

The Industry Foundation Classes (IFC)

a data model and open file format for BIM
- a data model and an open file format
- coming from STandard for the Exchange of Product model data (STEP)
- Each data model is mostly expressed as a schema in EXPRESS (XSD, OWL etc.)
- Published and maintained by buildingSMART

Selected IFC shortcomings

- Only used for exchange between BIM software that internally operate on proprietary data structures. (Ma and Sacks, 2016)

- Strong focus on geometry while the demands for knowledge capture are not adequately met. (Pauwels, 2018)

- No standardised query language to retrieve the data contained in a BIM model. (Zhang et al., 2018)

- EXPRESS is not widely used outside the engineering domain and not many general purpose tools support it. (Zhang et al., 2018)

Linked Data

  1. Use URIs as names for things
  2. Use HTTP URIs so that people can look up those names
  3. When someone looks up a URI, provide useful information, using the standards (RDF, SPARQL)
  4. Include links to other URIs. so that they can discover more things

                    # NOT LINKED DATA
                    <Eiffel_Tower> <is_a> <Place> .

                    # LINKED DATA IN N-TRIPLES SERIALISATION
                    <https://some-knowledge-graph.org/resources/Eiffel_Tower> 
                        <http://www.w3.org/1999/02/22-rdf-syntax-ns#type> <http://schema.org/Place> .

                    # LINKED DATA IN TURTLE SERIALISATION
                    @prefix inst:   <https://my-awesome-knowledge-graph.org/resources/> .
                    @prefix schema: <http://schema.org/> .

                    inst:Eiffel_Tower   a    schema:Place .

Research Task 3

Dealing with Evolving, Interdependent Design Properties

The Ontology for Property Management (OPM)

Resources should not be deleted

Three levels of property assignment

OPM: How it works

OPM and property reliability

Since the project knowledge and understanding increases over time it was essential that OPM has terminology to describe the reliability of some property
Classifying a property state as opm:Required means that it is not the actual property but instead a requirement for the future property value. In ECPPM paper it was demonstrated how this can be used to compare evolving client requirements with an evolving design
This illustrates the approach used for describing property requirements in the paper presented at ECPPM last fall
In engineering it is common practice to use assumptions in order to progress in situations where not all data input is available.
Even though OPM provides insights and allows dynamics to some extend it is not supposed to replace a good project execution plan. It is important to lock parts of the design at predefined milestones and in these cases OPM provides terminology to classify related properties as confirmed.

OPM and derived properties

OPM also has support for derived properties and a formal way of describing calculations that inferred them.
